### **계층 구조 아키텍처**

---

표현 → 응용 → 도메인 → 인프라스트럭처<br><br>

- 표현
    
    사용자의 요청을 받아 응용에 전달하고 응용으로부터 응답을 받아 클라이언트 또는 외부 시스템에 전달
  
    
- 응용
    
    유스케이스를 어떤 순서와 흐름으로 실행할지 결정하고, 여러 도메인 객체를 조합하여 전체 로직의 흐름을 조율합니다.
    
    즉, 비즈니스 규칙을 직접 수행하기보다는, 해당 로직의 실행을 도메인 모델에 위임하는 역할을 합니다.
    
- 도메인
    
    비즈니스 그 자체를 표현하는 영역으로, 규칙, 개념, 상태 변화, 불변 조건 등 핵심 비즈니스 로직을 포함합니다.
    
    상태와 행위를 함께 가지며, 비즈니스 규칙은 반드시 도메인 영역에 위치해야 합니다.
    
- 인프라스트럭처
    
    RDBMS 연동, 메시지 큐 송수신, 외부 API 호출 등 기술적인 구현 세부 사항을 담당하는 영역입니다.
    
    즉, DB 접근, 외부 API 연동, 메세지 브로커 등의 기술적인 구현을 합니다.<br><br>
    

> 도메인 모델 : 비즈니스 문제 영역을 개념적으로 표현한 전체 설계도
> 

코드일 수도 개념 다이어그램일 수도 있으며, 비즈니스 규칙과 개념의 집합을 의미합니다.<br><br>

> 도메인 객체 : 도메인 모델을 코드로 표현한 개별 구성 요소
> 

즉, 도메인 모델을 구성하는 실제 코드 단위 

ex ) Entity, Value Object ( 값 객체 ), 도메인 서비스 등 ..
